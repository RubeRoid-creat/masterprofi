# Тестовые аккаунты для мобильного приложения

## Аккаунты, созданные через seeds

В базе данных есть следующие тестовые аккаунты (создаются командой `npm run seed`):

### 1. Мастер (Master)
- **Email**: `master@masterprofi.ru`
- **Телефон**: `+79991234567`
- **Пароль**: `master123`
- **Роль**: master

### 2. Клиент (Client)
- **Email**: `client@masterprofi.ru`
- **Телефон**: `+79991234568`
- **Пароль**: `client123`
- **Роль**: client

### 3. Администратор (Admin)
- **Email**: `admin@masterprofi.ru`
- **Пароль**: `admin123`
- **Роль**: admin

## Как использовать в мобильном приложении

Мобильное приложение поддерживает вход по **email** или **телефону**.

### Примеры для входа:

**По email:**
- Email: `master@masterprofi.ru`
- Пароль: `master123`

**По телефону:**
- Телефон: `+79991234567` или `79991234567`
- Пароль: `master123`

## Демо-режим (без backend)

Если backend недоступен, мобильное приложение использует демо-режим:
- Можно ввести любой email/телефон и пароль
- Создается демо-пользователь с ролью `master`
- Данные не сохраняются на сервере

## Запуск seeds (если аккаунты не созданы)

```bash
cd backend
npm run seed:run
```

Это создаст тестовые аккаунты в базе данных.

## Проверка существующих аккаунтов

Если аккаунты уже созданы, вы можете проверить их через админ-панель или напрямую через базу данных.

## Важно!

- Убедитесь, что backend запущен и доступен по адресу `http://localhost:3000`
- Проверьте, что база данных подключена и seeds выполнены
- Если backend недоступен, приложение будет использовать демо-режим (любые данные для входа)
