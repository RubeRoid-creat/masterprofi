# App Store Submission Checklist

## Pre-Submission

### Assets Required

#### iOS App Store

- [ ] App Icon (1024x1024px PNG, no transparency)
- [ ] Screenshots for all required device sizes:
  - [ ] iPhone 14 Pro Max (1290x2796)
  - [ ] iPhone 11 Pro Max (1284x2778)
  - [ ] iPhone 8 Plus (1242x2208)
  - [ ] iPad Pro 12.9" (2048x2732)
  - [ ] iPad Pro 11" (1668x2388)
- [ ] App Preview Videos (optional)
- [ ] Promotional Text (170 characters max)
- [ ] Description (4000 characters max)
- [ ] Keywords (100 characters max)
- [ ] Support URL
- [ ] Marketing URL (optional)
- [ ] Privacy Policy URL

#### Google Play Store

- [ ] App Icon (512x512px PNG)
- [ ] Feature Graphic (1024x500px PNG)
- [ ] Screenshots:
  - [ ] Phone (1080x1920)
  - [ ] 7" Tablet (1200x1920)
  - [ ] 10" Tablet (1600x2560)
- [ ] Short Description (80 characters max)
- [ ] Full Description (4000 characters max)
- [ ] Privacy Policy URL

### Metadata

#### App Information

- [ ] App Name (30 characters iOS, 50 Android)
- [ ] Subtitle (iOS only, 30 characters)
- [ ] Category (Primary and Secondary)
- [ ] Age Rating
- [ ] Content Rating
- [ ] Pricing

#### Privacy & Compliance

- [ ] Privacy Policy URL (required)
- [ ] Privacy Policy accessible and up-to-date
- [ ] Data Collection disclosure (iOS)
- [ ] Privacy Manifest (iOS 17+)
- [ ] GDPR compliance (if applicable)
- [ ] CCPA compliance (if applicable)

### Technical Requirements

#### iOS

- [ ] Bundle ID configured
- [ ] App Store Connect app created
- [ ] Certificates and provisioning profiles
- [ ] TestFlight build uploaded
- [ ] App Store review information completed
- [ ] Version and build number set
- [ ] App Transport Security configured
- [ ] Required usage descriptions (permissions)
- [ ] App Store privacy questionnaire completed

#### Android

- [ ] Package name configured
- [ ] Google Play Console app created
- [ ] Signing key configured
- [ ] Target SDK version (34)
- [ ] Minimum SDK version (24)
- [ ] Version code and version name
- [ ] Content rating questionnaire completed
- [ ] Data Safety section completed

### Content Guidelines

- [ ] No placeholder content
- [ ] Real screenshots (not mockups)
- [ ] Accurate app description
- [ ] No misleading information
- [ ] Compliance with store guidelines

### Testing

- [ ] All features tested
- [ ] No crashes on launch
- [ ] TestFlight beta testing completed (iOS)
- [ ] Internal testing completed (Android)
- [ ] Critical bugs fixed
- [ ] Performance acceptable

## Submission Process

### iOS App Store

1. [ ] Archive build in Xcode or via EAS
2. [ ] Upload to App Store Connect
3. [ ] Fill all required metadata
4. [ ] Add screenshots and descriptions
5. [ ] Complete privacy questionnaire
6. [ ] Submit for review
7. [ ] Monitor review status

### Google Play Store

1. [ ] Build signed APK/AAB
2. [ ] Create release in Play Console
3. [ ] Upload APK/AAB
4. [ ] Fill store listing
5. [ ] Complete content rating
6. [ ] Complete data safety form
7. [ ] Submit for review
8. [ ] Monitor review status

## Post-Submission

### Review Follow-up

- [ ] Monitor review status
- [ ] Respond to reviewer questions
- [ ] Fix issues if rejected
- [ ] Resubmit if needed

### Launch Preparation

- [ ] Marketing materials ready
- [ ] Support team briefed
- [ ] Analytics configured
- [ ] Crash reporting active
- [ ] Update mechanism tested

## Release Checklist

- [ ] Release notes prepared
- [ ] Version number incremented
- [ ] Build number incremented
- [ ] All tests passing
- [ ] Documentation updated
- [ ] Changelog updated
